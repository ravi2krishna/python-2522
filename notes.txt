    -> Python functions can be defined to accept "parameters", 
        which act as placeholders for values that will be passed into the function 
        when it is called. These values are known as "arguments".

    -> We use parameters to make functions dynamic in nature 

    -> Based on arguments passed to functions, we got five types 

        -> Positional Arguments

        -> Keywords Arguments
        
        -> Default Arguments

        -> Arbitrary Positional Arguments

        -> Arbitrary Keywords Arguments
    
    -> Positional Arguments

        -> In Python, positional arguments are values passed to a function during 
        its call, where the assignment of these values to the function's parameters 
        is determined solely by their order (position) in the argument list. 
        The first argument in the function call is assigned to the first parameter 
        in the function definition, the second to the second, and so on. 

        -> Order Matters: If the order is incorrect, the function will receive the wrong values 
            for its intended parameters, potentially leading to errors or unexpected behavior.


    -> Keywords Arguments

        -> In Python, keyword arguments are a way to pass values to a function by explicitly 
        naming the parameters they correspond to, rather than relying on their position in the argument list. 
        This offers several advantages, primarily in terms of code readability and flexibility.
 

    -> Default Arguments

        -> In Python, default arguments allow you to define a function parameter with a pre-assigned value. 
        If a caller does not provide an argument for that parameter during a function call, 
        the default value is automatically used. If an argument is provided, it overrides the default.

        -> Placement Requirement: Default arguments must always be placed after any non-default (positional) 
            arguments in the function definition. 
            Attempting to place a non-default argument after a default argument will result in a SyntaxError.


    -> Arbitrary Positional Arguments

        -> In Python, arbitrary positional arguments allow a function to accept 
            a variable number of non-keyword arguments. 
            This is achieved by placing an asterisk (*) before a parameter name in the function definition. 
            
        -> This parameter will then collect all the extra positional arguments passed to the function 
            into a single tuple. 


    -> Arbitrary Keywords Arguments

        -> In Python, arbitrary keyword arguments, often represented as **kwargs, allow a function 
            to accept an unspecified number of keyword arguments. 
            These arguments are collected into a dictionary within the function, 
            where the argument names become the keys and 
            their corresponding values become the dictionary values.
        
        -> Include a parameter prefixed with two asterisks (**) in the function definition

    -> Types Of Functions 

        -> Built in functions 

            -> Pre defined functions, already part of standard python library 

            -> No need to explicitly create again using def keyword

            -> common utilities are generally predefined functions 

            -> examples 

                -> type, len, max, min, dir, id, input etc 

        -> Module Based Functions 

            -> Pre defined functions, given by vendors 

            -> Download modules or use system builtin modules 

            -> utilities designed for specific functionalities 

        -> User Defined Functions 

            -> User Defined functions are created by Developers, using def keyword

            -> Custom functionalities based on user given Requirements

            -> add(), emp_info() etc 

        
        -> Lambda Functions (Anonymous Functions) 

            -> A lambda function in Python is a small, anonymous function 
            defined with the "lambda" keyword. 
            
            -> Unlike standard functions defined with def, lambda functions are 
            restricted to a "single expression", which is "implicitly returned".

    -> OOP - Object Oriented Programming

        -> OOP, or Object-Oriented Programming, is a programming model 
        that organizes code around "objects" which are software entities 
        that contain both data (attributes) and the methods(functions) 
        that operate on that data. 

        -> Instead of a sequence of functions, OOP focuses on objects interacting 
            with one another, making code more modular, reusable and easier to maintain.

        -> Earlier 

            -> Writing Long List Of Instruction (task oriented)

        -> Moving Forward 

            -> Thinking in terms of real world entities (objects)

                -> Object - Data (attributes/variables)

                -> Object - Behavior (methods/functions)
        
            -> Writing code that follows Object Oriented Design 

                -> Focus is on objects and their relationships, not just functions 

                -> We don't think about steps, we think about "who does what"

        -> Four Pillars Of OOP 

            -> Inheritance 

                -> Using Inheritance We Get "Reusability"

            -> Encapsulation 

                -> Binding Data & Behavior Together 

            -> Abstraction

                -> Hiding Complexity, and show only what user needs 

            -> Polymorphism

                -> Extensibility, One Entity has multiple forms 

    Compare Functional vs OOP 

    ->                      Functional Style                        Object Oriented Style 

    -> Focus                Step by step functions                  Real World Objects 

    -> Data & Function      Separate                                Combined inside a class 

    -> Code Organization    One Big Script                          Multiple Reuseable Classes 

    -> Reusability          Harder                                  Easy as for Inheritance

    -> Maintenance          Complex as code grows                   More Cleaner 


    -> Functional Style Code 

        def student_attended_session(name):
            print(f"{name} Attended Session")

    -> Object Oriented Style Code 

        class Student:

            def __init__(self, name):
                self.name = name 

            def attended_session(self):
                print(f"{self.name} Attended Session")
